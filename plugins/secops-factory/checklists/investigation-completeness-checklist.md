# Investigation Completeness Checklist

**Weight:** 25%
**Purpose:** Verify all required investigation steps were performed and documented.

## Alert Metadata (5 items)

- [ ] Alert source/sensor documented
- [ ] Alert rule/signature ID captured
- [ ] Severity level stated (Critical/High/Medium/Low)
- [ ] Detection timestamp recorded
- [ ] Event occurrence time noted

## Network/Host Identifiers (5 items)

- [ ] Source IP/hostname documented
- [ ] Destination IP/hostname documented
- [ ] Protocol and port information specified
- [ ] Asset type identified (server, workstation, ICS device, network equipment)
- [ ] Asset criticality assessed (Critical/High/Medium/Low)

## Investigation Steps (5 items)

- [ ] Relevant logs collected from appropriate sources
- [ ] Correlation performed with related events
- [ ] Historical context researched (30/60/90 day patterns)
- [ ] Asset ownership confirmed
- [ ] Business context obtained

## Evidence and Analysis (4 items)

- [ ] Technical analysis performed (protocol, attack vector, IOC)
- [ ] Threat intelligence consulted (if external IP involved)
- [ ] Evidence documented with timestamps
- [ ] Chain of custody maintained (local save before JIRA update)
