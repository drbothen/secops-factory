## Description

The monitoring-loop is the secops-factory's autonomous patrol mode. It runs
unattended on a schedule (cron or Task Scheduler) and performs full Tier-1 alert
triage using a 9-stage pipeline (Stage 0 per-org/sensor health check, then 8
per-alert stages). For each registered org, it ingests new events since the last
watermark, validates and categorizes them, enriches via prism ThreatIntel/NVD
and web sources (Perplexity + Tavily), scores by asset criticality and rule
fidelity, produces one of four canonical dispositions (TP/FP/BTP/Indeterminate),
executes a Jira-first ticket action, and persists an ICD-203-grade defensible
verdict record. Hard floors unconditionally prevent autonomous action on
high-severity, critical-asset, and Indeterminate categories regardless of
configuration. All Jira write operations require a disposition-guard-issued marker
(D-DEC-001).
